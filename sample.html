Introduction to Graph Representation
What is Graph Representation?

Storing a graph in a computer system using a data structure, so it can be manipulated algorithmically. Just as an array is represented as int arr[n] in C++, a graph needs its own structure.

Two primary methods: Adjacency Matrix and Adjacency List
A graph consists of vertices (V) and edges (E) between them
Tree is a special graph: n nodes, n−1 edges, no cycles, one root
🔀
Graph Types
Undirected Unweighted

Edges have no direction and no weight. Matrix is symmetric. 0 or 1 values only.

Undirected Weighted

Edges have no direction but carry numeric weights. Matrix stores the weight value.

Directed Unweighted

Edges have direction (one-way). Matrix is asymmetric — only (i,j) is set.

Directed Weighted

Edges have both direction and weight. Most complex. Used for road networks.

📊
Adjacency Matrix
A V×V 2D array. Cell [i][j] = 1 (or weight) if edge exists, 0 otherwise
For undirected graphs: symmetric — [i][j] always equals [j][i]
Space: O(V²) regardless of number of edges — wasteful for sparse graphs

C++ — Undirected Unweighted 
int v, e;
cin >> v >> e;

vector<vector<int>> mat(v, vector<int>(v, 0));

for (int i = 0; i < e; i++) {
    int u, w;
    cin >> u >> w;
    mat[u][w] = 1;
    mat[w][u] = 1;  // undirected: both sides
}
📋
Adjacency List
Array of vectors — each vertex holds a list of its neighbors
Space: O(V + E) — much better for sparse graphs
Weighted: store pair<int,int> (neighbor, weight) in each list
C++ — Unweighted 
vector<vector<int>> adj(v);

for (int i = 0; i < e; i++) {
    int u, w;
    cin >> u >> w;
    adj[u].push_back(w);
    adj[w].push_back(u);  // undirected
}
C++ — Weighted 
vector<vector<pair<int,int>>> adj(v);  // {neighbor, weight}

for (int i = 0; i < e; i++) {
    int u, w, wt;
    cin >> u >> w >> wt;
    adj[u].push_back({w, wt});
    adj[w].push_back({u, wt});
}

Head-to-Head Comparison
📊 Adjacency Matrix
Space O(V²)
Edge lookup O(1)
Edge delete O(1)
Best for Dense graphs
📋 Adjacency List
Space O(V+E)
Edge lookup O(degree)
Edge delete O(degree)
Best for Sparse graphs
💡 Real-world tip

Adjacency List is used in almost all competitive programming and real-world applications (social networks, road maps) because real graphs are sparse — Facebook users average ~300 friends out of billions of possible connections.

Key Terms
Vertex (V)
A node in the graph. Identified by a number (0 to V-1).
Edge (E)
Connection between two vertices. Can have direction and/or weight.
Degree
Number of edges connected to a vertex. Directed: in-degree + out-degree.
Sparse Graph
Few edges relative to V². Prefer adjacency list.
Dense Graph
Edges approach V². Adjacency matrix becomes viable.
In-degree
Edges coming IN to a node (directed graphs). High out, low in = spam node.

Is the adjacency matrix symmetric for directed graphs?
No. Only mat[u][v] is set for edge u→v. mat[v][u] remains 0 unless there's a reverse edge.
